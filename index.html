Sobre Este Projeto

Projeto de portfólio com foco total em aprendizado e prática de front-end, desenvolvido para consolidar conceitos de organização de código, arquitetura de software e consumo de dados externos em aplicações web modernas.

O projeto utiliza Vite como ferramenta de build e dev server, com JavaScript Vanilla, HTML, CSS e Bootstrap na interface. A organização dos estilos segue a metodologia BEM, garantindo padronização, legibilidade e escalabilidade do CSS. A estrutura do código é inspirada em princípios de DDD (Domain-Driven Design) e Clean Architecture, com separação clara entre camadas de domínio, aplicação, infraestrutura e UI.

A arquitetura adota padrões como Use Cases e Repository Pattern, promovendo baixo acoplamento, separação de responsabilidades e facilidade de evolução do projeto. Essa abordagem permite trocar fontes de dados, alterar a interface ou expandir funcionalidades sem impacto direto nas regras centrais da aplicação.

Os dados de campeões e skins foram inicialmente obtidos a partir do Data Dragon (Riot Games), servindo como base oficial para nomes, identificadores e estrutura das informações. A partir desses dados, foi criado um fluxo automatizado de organização e processamento para padronização e uso interno no projeto.

As imagens são servidas via Cloudinary, simulando um cenário real de uso de CDN e integração com serviços externos. Isso reforça boas práticas de modularização, isolamento da infraestrutura e consumo de recursos remotos. Para melhorar a experiência de seleção, o projeto utiliza um select customizado com Select2, permitindo busca rápida e interação mais eficiente com a lista de personagens.

Como funcionalidade principal, o usuário pode selecionar um personagem em uma lista interativa (com nome e ícone), e a aplicação carrega dinamicamente um carrossel com todas as skins correspondentes ao personagem escolhido. A troca de personagem atualiza todo o conteúdo relacionado, como imagens, textos e estados visuais, mantendo a experiência simples, rápida e orientada a dados.
